1 ER図

ER図 必須
テーブル間のリレーション図（テキストベースまたは画像で記載）
【ER図】

[USERS]
  ID (PK)
  USERNAME
  PASSWORD_HASH
  ROLE
  ENABLED
  CREATED_AT
  UPDATED_AT

[CONTACT_INQUIRIES]
  ID (PK)
  CONTACT_METHOD
  CREATED_AT
  EMAIL
  ENGLISH_PAGE
  INQUIRY_TYPE
  MESSAGE
  NAME
  PLAN
  REQUEST_TYPES
  PREFERRED_DATE
  PHONE_NUMBER
  DELETED_AT

【リレーション】
・USERS：管理者および管理対象ユーザー情報を管理
・CONTACT_INQUIRIES：公開サイトから送信されたお問い合わせ情報を管理
・現状は USERS と CONTACT_INQUIRIES を直接関連付けない独立テーブル構成

2 テーブル定義

テーブル名：USERS 必須
テーブルの目的：管理者および管理対象ユーザー情報を管理
カラム名データ型NULLデフォルト制約説明
IDBIGINTNO-PK主キー
USERNAMECHARACTER VARYING(50)NO-UNIQUEユーザーID
PASSWORD_HASHCHARACTER VARYING(255)NO--パスワードハッシュ
ROLECHARACTER VARYING(20)NO--権限
ENABLEDBOOLEANNO--有効フラグ
CREATED_ATTIMESTAMPNO--作成日時
UPDATED_ATTIMESTAMPNO--更新日時
テーブル名：CONTACT_INQUIRIES 必須
テーブルの目的：公開サイトから送信されたお問い合わせ情報を管理
カラム名データ型NULLデフォルト制約説明
IDBIGINTNO-PK主キー
CONTACT_METHODCHARACTER VARYING(50)YES--連絡方法
CREATED_ATTIMESTAMP(6)NO--受信日時
EMAILCHARACTER VARYING(255)NO--メールアドレス
ENGLISH_PAGEBOOLEANNO--英語ページ経由フラグ
INQUIRY_TYPECHARACTER VARYING(20)NO--問い合わせ種別
MESSAGECHARACTER VARYING(4000)NO--お問い合わせ内容
NAMECHARACTER VARYING(120)NO--名前
PLANCHARACTER VARYING(120)YES--希望プラン
REQUEST_TYPESCHARACTER VARYING(255)YES--希望内容
PREFERRED_DATEDATEYES--希望日
PHONE_NUMBERCHARACTER VARYING(30)YES--電話番号
DELETED_ATTIMESTAMPYES--削除日時

本システムのDBスキーマ変更管理は Flyway で実施する。flyway_schema_history テーブルは Flyway の内部管理用のため、本設計書の業務テーブル定義対象外とする。

インデックス一覧 必須
テーブル名インデックス名対象カラム種類設定理由
