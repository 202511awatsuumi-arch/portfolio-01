1 システム概要

システムの目的
このシステムが解決する課題と提供する価値

「香り結び」の管理システムは、店舗管理者が予約希望や各種お問い合わせを一元管理し、 確認・編集・削除や
ユーザー管理を行うことで、対応漏れ防止と顧客対応の品質向上、運営効率化を実現するためのシステムです。

4 機能詳細仕様

ログイン

機能名：ログイン
機能ID：F-001

管理者がユーザーIDとパスワードを入力して認証を行い、管理画面へログインする機能。

【画面レイアウト】
・管理ログインタイトルを表示
・ユーザー名入力欄、パスワード入力欄を配置
・ログインボタンを配置
・エラー時はメッセージを表示

【処理フロー】
1. ユーザー名とパスワードを入力
2. ログインボタン押下
3. システムが認証処理を実行
4. 成功時は管理画面へ遷移
5. 失敗時はログイン画面を再表示

【正常時の動作】
・管理者ユーザー一覧画面を表示

【エラー時の動作】
・入力不備または認証失敗時、エラーメッセージを表示

【画面遷移先】
・成功時：管理者ユーザー一覧画面
・失敗時：管理者ログイン画面

入力項目一覧
項目名 データ型 必須 バリデーション 備考
ユーザー名 文字列 必須 未入力不可 認証時に使用
パスワード 文字列 必須 未入力不可 認証時に使用

ログアウト

機能名：ログアウト
機能ID：F-002

管理者が管理画面上のログアウト操作を行い、セッションを終了してログイン画面へ戻る機能。

【画面レイアウト】
・管理画面のヘッダーまたはメニュー内にログアウト導線を配置
・ログアウト実行後はログイン画面を表示
・ログイン画面上にログアウト完了メッセージを表示

【処理フロー】
1. 管理者がログアウトを押下
2. システムがセッションを無効化
3. 認証情報を破棄
4. ログイン画面へ遷移
5. ログアウト完了メッセージを表示

【正常時の動作】
・ログアウト成功後、管理者ログイン画面を表示する
・ログイン画面に「ログアウトしました。」を表示する

【エラー時の動作】
・通常はログイン画面へ遷移する
・システムエラー時は標準エラー処理に従う

【画面遷移先】
・遷移先：管理者ログイン画面

入力項目一覧
項目名 データ型 必須 バリデーション 備考
なし - - - ボタン押下で実行

機能名：ユーザー一覧表示
機能ID：F-003

管理者が登録済みユーザー情報を一覧で確認する機能。

【画面レイアウト】

3. 一覧テーブルに表示する
4. 必要に応じて編集・削除・新規登録へ遷移する

【正常時の動作】
・登録済みユーザー情報を一覧表示する
・ページ単位で表示する

【エラー時の動作】
・データ取得失敗時はエラーメッセージを表示する

【画面遷移先】
・新規登録押下：管理者ユーザー登録画面
・編集押下：管理者ユーザー編集画面
・削除押下：管理者ユーザー一覧画面

項目名 データ型 必須 バリデーション 備考
なし - - - 一覧表示機能

ユーザー新規登録

機能名：ユーザー新規登録
機能ID：F-004

管理者が新しいユーザー情報を入力し、登録する機能。

【画面レイアウト】
・画面上部にタイトルを表示
・ユーザー名入力欄を配置
・メールアドレス入力欄を配置
・パスワード入力欄を配置
・パスワード確認入力欄を配置
・権限選択欄を配置
・登録ボタン、戻るボタンを配置
・エラー時は入力欄付近または画面上部にメッセージを表示

【処理フロー】
1. 管理者が登録画面を開く
2. 必要項目を入力する
3. 登録ボタンを押下する
4. システムが入力チェックを行う
5. 問題がなければユーザー情報を登録する
6. 登録完了後、ユーザー一覧画面へ遷移する

【正常時の動作】
・入力内容が正しい場合、ユーザー情報を登録する
・登録完了後、管理者ユーザー一覧画面を表示する

【エラー時の動作】
・必須未入力、形式不正、確認用パスワード不一致時はエラーメッセージを表示する
・登録失敗時は入力画面を再表示する

【画面遷移先】
・登録成功時：管理者ユーザー一覧画面
・エラー時：管理者ユーザー登録画面
・戻る押下時：管理者ユーザー一覧画面

入力項目一覧
項目名 データ型 必須 バリデーション 備考
ユーザー名 文字列 必須 未入力不可 ログイン時に使用
メールアドレス 文字列 必須 未入力不可 連絡先として使用
パスワード 文字列 必須 未入力不可 認証時に使用
パスワード確認 文字列 必須 未入力不可、一致必須 パスワード確認用
権限 文字列 必須 未選択不可 管理者/一般ユーザー区分

ユーザー削除

機能名：ユーザー削除
機能ID：F-006

管理者が登録済みユーザー情報を削除する機能。

【画面レイアウト】
・ユーザー一覧テーブルの各行に削除ボタンを配置
・必要に応じて確認メッセージを表示
・削除後は一覧画面を再表示する

【処理フロー】
1. 管理者がユーザー一覧画面を開く
2. 削除対象ユーザーの削除ボタンを押下する
3. システムが対象ユーザー情報を削除する
4. 削除後、ユーザー一覧画面を再表示する

・対象ユーザー情報を削除する
・削除後、最新状態の管理者ユーザー一覧画面を表示する

【エラー時の動作】
・削除対象が存在しない場合や削除処理失敗時はエラーメッセージを表示する
・削除できない場合は一覧画面を再表示する

【画面遷移先】
・削除実行後：管理者ユーザー一覧画面

セキュリティ要件
認証・認可・データ保護に関する要件

・認証方式：Spring Securityによるフォーム認証を採用し、管理画面は認証済みユーザーのみアクセス可能とする。
・パスワードの暗号化方式：パスワードは BCrypt によりハッシュ化して保存し、平文では保持しない。
・CSRF対策：Spring Security の CSRF 保護を有効とし、状態変更を伴うリクエストでは CSRF トークンを検証する。
・XSS対策：Thymeleaf のHTMLエスケープを利用し、入力値表示時はエスケープ処理を行うことでスクリプト実行を防止する。
・SQLインジェクション対策：Spring Data JPA / Repository を利用し、プレースホルダ付きクエリでDBアクセスを行うことで不正なSQL実行を防止する。
・セッション管理：ログイン成功時にセッションを開始し、ログアウト時はセッションを無効化して認証情報を破棄する。
